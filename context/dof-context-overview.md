# DOF 公司概況

## dreamoffish（DOF）係咩

dreamoffish 係一間香港嘅 **video production company**，主要做 corporate video——即係幫企業客戶拍片。

### 做咩類型嘅片
| 類型 | 例子 |
|------|------|
| Brand Film | 企業形象片、品牌宣傳 |
| Social Media Content | 社交媒體短片 |
| Event Coverage | 活動紀錄 |
| Internal Training | 內部培訓影片 |
| Campaign | 宣傳活動片 |
| Fundraising | 籌款活動影片 |
| Documentary | 紀錄片 |

### 典型客人
| Client Type | 例子 |
|-------------|------|
| Government | 政府部門投標項目 |
| Corporate | CHANEL、CLP（中電）、EMSD（機電署）、HSUHK |
| NGO / Faith | 慈善機構、宗教團體 |

### 公司規模
- 10 人左右嘅 team
- 冇 in-house shooting crew——拍攝全部用 freelancers
- 後期（剪接、motion graphics）有 in-house team

## 核心流程

```
Client Brief → Script → Pitch Deck → Shoot → Post-Production → Delivery
```

- 導演由頭帶到尾（Creative + PM）
- 後期組 follow Google Calendar 嘅 milestone 排期做嘢
- Client feedback 通常經 WhatsApp 嚟，導演整理後 share 去 Discord

詳細見 `production-pipeline.md`、`client-feedback-workflow.md`

## 而家嘅 Pain Points（背景知識）

### 1. Calendar 資訊唔齊全
唔係每個導演都完整 mark milestone。後期組 plan workload 嘅時候可能唔夠資訊。

### 2. Admin 工作冇人做
冇咗 Marketing 同事之後，quotation 全部 Ki 自己做，job list 維護由 Kary + Sohling 人手處理。

### 3. 文件化唔夠
公司好多嘢靠口頭傳承。新人 onboarding 冇 standard process。

### 4. Workflow 紀律唔一致
有導演嚴格跟 workflow（Calendar、Google Doc、Discord sharing），有導演唔跟。
系統設計方向：降低 friction + Ki endorse → 令全 team 跟從。

## Mugi 喺 DOF 嘅定位

Mugi 係 DOF team 嘅 **production operations assistant**——唔係 general chatbot。

核心價值：
1. **降低 Calendar 管理嘅 friction** — 自然語言操作，唔使人手逐個改
2. **回答 DOF workflow 問題** — 新同事或唔熟流程嘅人可以問
3. **Enforce naming convention** — Mugi 建立嘅 events 一定跟標準格式
4. **提供 timeline 估算** — 根據標準工時回答「幾時交片」嘅問題

Mugi **唔做**嘅嘢：
- 唔做 general chatbot 嘅事（翻譯、寫 email、解釋技術概念）
- 唔做新 job 建立（呢個經 Project Portal form）
- 唔做 creative 工作

詳細 scope 見 CLAUDE.md 嘅 Role Boundaries。
